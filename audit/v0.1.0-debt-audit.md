## Known debt items

### D-001 — `node:sqlite` is experimental (Node.js v22)
**File:** `src/__tests__/helpers.ts`  
**Risk:** Low. Test-only; not in production code path. Node.js 23+ will stabilise the API. No action needed until Node.js LTS changes.

### D-002 — Rate-limiter bindings are mocked as always-pass stubs
**File:** `src/__tests__/helpers.ts`  
**Risk:** Low. Unit tests do not exercise the rate-limit rejection path; that path is tested by the real Cloudflare runtime only. Acceptable for v0.1.0 — integration tests against a deployed staging worker would cover this.

### D-003 — No KV warm-up on `wrangler dev` cold start
**File:** `src/scheduler.ts`  
**Risk:** Low. The scheduled handler populates KV; `wrangler dev` triggers it on first `curl`. Documented workaround: `wrangler dev --trigger scheduled` once per session.

### D-004 — `listEuAlternatives` SQL does full table scan on `category`
**File:** `src/db.ts`  
**Risk:** Negligible at 35 providers. `idx_category` index exists. No action needed until provider count exceeds ~10 000.

### D-005 — `scan_stack` makes N individual D1 queries (one per provider name)
**File:** `src/mcp.ts`  
**Risk:** Low at current scale. KV cache hit path is O(N) parallel KV reads, which is fast. D1 fallback is N sequential queries — acceptable up to ~50. If stack scan becomes a hot path post-launch, batch with `WHERE name IN (...)`.
